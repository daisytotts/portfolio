# Healthcare Readmission Risk Analysis

## Project Summary

This project explores how healthcare operations understanding and data analytics can work together to identify hospital readmission patterns.

The goal was to build a small end-to-end healthcare analytics workflow using Python and SQL. The project focuses on 30-day readmissions and shows how structured data can support earlier visibility, better discharge planning, and stronger operational decision-making.

This project uses simulated/public healthcare-style admission data for portfolio demonstration. No real patient data is used.

## Why This Project Matters

Hospital readmissions are not only clinical events. They can also show system-level problems such as discharge planning gaps, limited follow-up coordination, workflow pressure, and lack of early visibility.

From my healthcare background, I know that teams often work under pressure and may not always have time to see larger patterns behind repeated patient admissions. This project was built from that perspective.

## Business and Operational Problem

The project asks one main question:

**Can structured hospital data help identify patients at higher risk of readmission before the problem repeats itself?**

The focus is not only prediction. The real value is decision support.

## Project Workflow

Raw hospital CSV data → Python preprocessing → Cleaning and transformation → SQL storage → Readmission analysis → Operational insights

## What I Built

* Cleaned hospital admission data using Python
* Standardized admission and discharge dates
* Created a 30-day readmission indicator
* Analyzed repeated admission patterns
* Wrote SQL queries to identify patients with higher readmission risk
* Created visual summaries for operational decision support
* Connected technical analysis with real healthcare workflow problems

## Key Insights

* Patients with repeated previous admissions showed higher readmission risk.
* Short discharge-to-readmission gaps may point to workflow or follow-up gaps.
* Readmissions are not always isolated events. They can reflect larger system patterns.
* Structured data can help healthcare teams move from reactive work to earlier visibility.
