# Business KPI Dashboard & Data Modeling

## Project Summary

This project shows how I would build a clean business reporting system from raw data to management insights.

Nova Retail Group is a fictional mid-sized retail company with sales across regions, product groups and customer segments. The company had data, but the reporting process was messy. Sales, customer and product data were stored in different CSV files, and reports were prepared manually in Excel.

My goal was to create one clean reporting foundation using Python, SQL and Power BI.

## Business Problem

The company needed better reporting because:

* Reports took too much time to prepare
* Different teams used different spreadsheets
* KPI definitions were not always clear
* Manual Excel work created room for mistakes
* Managers could not easily compare regions, products and customer groups

The company did not need more spreadsheets. It needed a better reporting structure.

## Project Workflow

Raw CSV files → Python cleaning → SQL database → Star schema data model → Power BI dashboard → Business insights

## What I Built

* Cleaned and prepared raw business data with Python
* Created calculated fields such as revenue, cost, profit and profit margin
* Designed a relational SQL data model
* Built a star schema with one fact table and dimension tables
* Created business KPI calculations
* Designed a Power BI dashboard for management reporting
* Turned the dashboard results into business recommendations

## Main KPIs

* Total Revenue
* Total Profit
* Profit Margin
* Number of Orders
* Average Order Value
* Customer Count

## Business Insights

The dashboard helped answer questions such as:

* Which regions perform best?
* Which product categories bring high revenue but low profit?
* Which products drive most of the business?
* Which customer segments create the highest value?
* Where should management focus next?

## What Makes This Project Different

I did not build only a dashboard. I built the full reporting process behind it.

My healthcare operations background also shaped how I worked on this project. In healthcare, small mistakes can create serious problems. I bring the same mindset into data work: clean data first, clear structure second, dashboard last.
